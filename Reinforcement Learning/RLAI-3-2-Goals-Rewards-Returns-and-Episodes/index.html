(draft)'RL-Goals, Rewards, Returns and Episodes'

Posted on 2020-04-15 Edited on 2020-11-08 In category Disqus:

Keywords:

We have introduced the concepts about agent and environment in reinforcement learning in the post ‘The Agent-Environment interface’. In this post, we introduce another 4 concepts and some relative ideas about them. They are:

  1. Goals,
  2. Rewards,
  3. Returns,
  4. And Episodes.

Goals and Rewards

First, we should verify the concept – goals. Because, the goal of agent and the goal of algorithm designers are different. We, the algorithm designers, want to solve some practical task, like building a smart system to collect information that we concern. In this task, our goal is to collect information, and we do not do that by ourself fot that is tedious. And we make a smart system to do that. However, the information is nothing but some bits for the system, its goal is something else we give it, such as maximizing some objective functions. The goal of the system, is the goal of the agent when we talk about reinforcement learning.

In the reinforcement learning, the purpose or goal of agent is formalized in terms of a special signal, we call it reward. The reward is a number, denoted as:
$$
R_t\in \mathbb{R}
$$

where the subscript $t$ means the time or step. This signal, the reward, is the key point through which the evironment interact with the agent(on the other direction the agent interact with the environment by actions).

Informally, the agent’s goal is to maximize the total amount of reward it recieves. It emplies that the goal is to maximize cummulative reward in the long run but not the immediate reward.

All above description consiste the reward hypothesis:

“That all of what we mean by goals and purposes can be well thought of as the maximization of the expected value of the cummulate sum of a recieved scalar signal(called reward)”[^1]

Converting goal to reward is limiting in the reinforcement learning but flexible and wildly applicable. For example, we design a robot to collect empty cans, and we assign reward $-1$ to the action that the robot collects a book all other thing but not empty cans, or a reward $+1$ to the action it collects an empty can, or reward $0$ when it is searching or doing other things.

The rewards represet what we want accomplished. We provide rewards to the agent, and the agent would maximize its cumulative rewards by optimizing its action or policy. What the agent learned in this process is our goals.

However, goals is not the elements to represent prior knowledge, such as how to achieve what we want the agent to do. These prior knowledge can be combined into initial policy and intitial condition.

Unlike other machine learning tasks, seperating reinforcement learning problems into some subgoals are not a good idea. Because the subgoals may lead the agent to another deriction but not the goal.

Reward is a way in reinforcement learning tasks through which we communicate to the robot what you want to achieve.

Returns and Episodes

Goal of agent is maximizing the cumulative reward it recieves in the long run. However, the long run cumulative rewrd is not defined yet. And the formally definition of cumulative rewrds. The rewards from $t$ are:
$$
R_{t+1},R_{t+2},\cdots,R_{T}\tag{1}
$$

The agent seek to maximize the expected return:
$$
G_t\doteq R_{t+1}+R_{t+2}+\cdots+R_{T}\tag{2}
$$

Where $T$ is the final step. However, this $T$ can a number or $\infty$.

Episodes

When $T$ is finite, that means the sequence has an end. The agent-environment interaction breaks naturally into subsequences and the subsequence is called episodes. So there is a terminal state of each episode. The next episode start independently of previous episode. To a same problem, there can be many different episodes with different final cumulative rewards but with the same terminal state. And to episode tasks, $T$ in each episode is a random variable. Nonterminal states are alway denoted as $\mathcal{S}$ and the states containing terminal states and nonterminal states are denoted as $\mathcal{S}^+$.

Continuing Tasks

When $T$ is infinite, the task could not stop naturely. This is common in on-going process-control task:

$$
G_t\doteq \sum_{i=t+1}^{\infty}R_{i}\tag{3}
$$

In this kind of task the cumulative rewards could be infinity easily.

Discounting

The return could be described in equation (2). But that is not the only form. We can design a return function of reward of each step(time) to different tasks. For example, we define the ‘expected discounted return’:
$$
\begin{aligned}
G_t&\doteq R_{t+1}+\gamma R_{t+2}+\cdots+\gamma^{k}R_{t+k+1}\
&=\sum_{k=0}^{\infty} \gamma^kR_{t+k+1}
\end{aligned}\tag{4}
$$

where $0\leq \gamma \leq 1$ is called discount rate. The value of future rewards could be:

  1. when $\gamma<1$ , the infinite sum in equation (4) has a finite value as long as the reward sequence ${R_k}$ is bounded
  2. when $\gamma =0$ the agent is myopic that the agent only concern immediate rewards.
  3. when $\gamma \to 1$, the agent take future rewards into account more strongly. This time the agent is farsight.

Returns at successive time steps are related to each other. And it is important for the theory and algorithm of reinforcement learning:

$$
\begin{aligned}
G_t&\doteq R_{t+1}+\gamma R_{t+2}+\gamma^2 R_{t+3}+\gamma^3 R_{t+4}+\cdots\
&=R_{t+1}+\gamma (R_{t+2}+\gamma R_{t+3}+\gamma^2 R_{t+4}+\cdots)\
&=R_{t+1}+\gamma G_{t+1}
\end{aligned}\tag{5}
$$
where $t<T$. Equation (5) is valid even if termination occurs at $t+1$ if we define $G_T=0$. If the reward is constant $+1$, we have:
$$
G_t=\sum^{\infty}_{k=0}\gamma^k=\frac{1}{1-\gamma}\tag{6}
$$

Unified Notion for Episodic and Continuing Tasks

Episodic task is easier than continuing tasks. And they have different notations. But we want to unify them for we usually talk about them sumultaneously. To represent the continuing task, we add absorbing state, whcih transitions only to itself with reward $0$, to episodic task:

Discounting also works in this form:
$$
G_t\doteq \sum_{k=t+1}^{T}\gamma^{k-t-1}R_k
$$

including $T=\infty$ or $\gamma=1$(but not both).

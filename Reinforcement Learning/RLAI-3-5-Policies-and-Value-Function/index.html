(draft)Policies and Value Function

Posted on 2020-04-18 Edited on 2020-11-08 In category Disqus:

Abstract: Beside the concepts we have talked about, like agent, environment, goals and rewards, value and value function are another two common words in reinforcement learning. Almost all reinforcement learning algorithms involve estimating value functions. Two forms of value function are: 1. function of states: how good the agent would be from a given state. 2. function of action(or 'state-action' pairs): how good an agent would be when it perform an action(i.e. \(a\)) at a given state(i.e. \(s\)).

'How good' can be measured in many ways, but in reforcement learning it is always defined by the expected future rewards, or equally the expected return. However, before we define the value function, we should know how to select an action. Because the action is the only way through which the agent interact with the environment, and the future rewards are fully decided by the environment. In one word, value function is defined of rewards, and rewards is the reaction of the environment to a certain action of the agent.

Policy

The policy is how to select an action for a state. We know the state is the information the agent can get from current environment. And the action is produced by the agent basing on the state it is faceing to. For example, we are the agent, and the environment is the weather outside the house. We can not see what the weather is like outside for the curtain was draw. We can only hear like it is raining(this is the description of state, the information the agent could collect from environment). Then we should decide what to do: 1. stay at home 2. go to work on foot with raincoat 3. go to work by car without raincoat or umbrella

And the reward of action (1) and of action (2) and (3) are different obviously. And the difference of the reward of action (2) and of action (3) might be slight. However, this is not we should concern now. What we should do is just to select an action of these three. This is the policy, and it is denoted in a probability form:

\[ \pi(a|s)\tag{1} \]

This is a conditional probabilty of action \(a\) given state \(s\) which means how to select an action of a given state. And reinforcement learning method specify how the agent's policy is changed as a result of it experience.

Value Function

Once we have the definition of policy, we can have a close look at the valur function.

Value Function of State

The value function of state \(s\) under a policy \(\pi\) is denoted as: \[ v_{\pi}(s)\tag{2} \]

For the MDP, the value function of state \(s\) is:

\[ \begin{aligned} v_{\pi}(s)&\doteq \mathbb{E} [G_t|S_t=s]\\ &=\mathbb{E}_{\pi}[\sum_{k=0}^{\infty}\gamma^kR_{t+k+1}|S_t=s] \end{aligned}\tag{3} \]

for all \(s\in \mathcal{S}\) and the value of terminal state is setted to \(0\). Equation (3) is the state-value function for policy \(\pi\).

Value Function of Action

Similarly, the value fo taking action \(a\) in state \(s\) under a policy \(\pi\) is denoted as:

\[ q_{\pi}(s,a)\tag{4} \]

It represents the expected return starting from state \(s\), taking action \(a\). And its definition is:

\[ \begin{aligned} q_{\pi}(s,a)&\doteq \mathbb{E}_{\pi}[G_t|S_t=s,A_t=a]\\ &=\mathbb{E}[\sum_{k=0}^{\infty}\gamma^kR_{t+k+1}|S_t=s,A_t=a] \end{aligned}\tag{5} \]

Equation (5) is the action-value function for policy \(\pi\).

Value function is widely used in reinforcement learning. And it can be estimated from experience. For example, in the following posts we will introduced: 1. Monte Carlo method which estimates the value by averaging returns 2. and parameterized function approximator which estimates the value by approximating returns.

Value functions statisfy recuresive relationships. For any policy \(\pi\) and any state \(s\), following relation hold:

\[ \begin{aligned} v_{\pi}(s)&\doteq \mathbb{E}_{\pi}[G_t|S_t=s]\\ &=\mathbb{E}_{\pi}[R_{t+1}+\gamma G_{t+1}|S_t=s]\\ &= \sum_a\pi(a|s)\sum_{s'}\sum_{r}p(s',r|s,a)[r+\gamma\mathbb{E}_{\pi}[G_{t+1}|S_{t+1}=s']]\\ &= \sum_a\pi(a|s)\sum_{s',r}p(s',r|s,a)[r+\gamma v_{\pi}(s')] \end{aligned}\tag{6} \]

for all \(s\in mathcal{S}\), where \(a\in \mathcal{A}, r\in mathcal{R},s\in\mathcal{S}\text{ or }\mathcal{S^+}\). \(p(s',r|s,a)\) in equation (6) is the dynamics of the environment under MDP and the whold process is summing over \(a\), \(s'\) and \(r\). This is called Bellman equation for \(v_{\pi}\). And it represents the relation of current value of state and the value of its successor states.

Backup Diagram

Backup diagram is a powerful tool to illustrate the process of Bellman equation above and other similar process:

The open blue circle are states, and black solid circles represent action, \(\pi\), \(r\) and \(p\) are policy, reward and dynamics respectively. And it reprecents the action is selected basing on probability \(\pi\) and the next state is produced by dynamics who is a probability of \(s\), \(a\), \(r\) and \(s'\). So the expectation return of state \(s\) can be calculated by two stages: 1. the first one is expectation of a certain action \(a_i\) by the row of \(s'\) : \(\sum_{s',r}p(s',r|s,a)[r+\gamma v_{\pi}(s')]\) 2. the second one is expectation of state \(s\) from stage 1: \(\sum_a\pi(a|s)\cdot{\text{'result of stage 1'}}\)

and these stages form the equantion (6). For the whole process starts at bottom which represent the future step, it is caled backup diagram.

To finite MDP, there is unique solution to Bellman equation. For we can list \(n\) equations who have \(n\) unknowns(it can solve by already known methods). And it is the basis to compute \(v_{\pi}\)

Backup diagram would be used in the following posts, because of its simplicity. It is not a transition graph for it just part of the transition.

from Supervised Learning to Reinforcement learning[1]

Some supervised learning algorithms like traditional machine learning algorithms, like linear regression, linear classification, and other more complex algorithms and deep learning, like neural networks are all based on training data who acts as a teacher told the algorithms what is correct and led the algorithms to some specified form. And the teacher gave the ‘correct’ answers no matter what the algorithm outputted.

However, reinforcement learning works in another way, the training data does not give any correct answer to an algorithm after its action. But it just evaluates the action of the algorithm. Then the algorithm modified itself for getting higher rewards given by the teacher(the result of the evaluation of the teacher). This is the main distinction between supervised learning and reinforcement learning.

A full reinforcement learning is too complex for us to begin with. So, we use a more simplified problem which is non-associative to have a first look at this kind of problem. The non-associative problem is the problem whose most prior work, involving evaluative feedback, has been done before we work on it. A well known non-associative task is the ‘kkk-armed bandits’.

kkk-armed Bandit

A kkk-armed bandit problem can be simply described as such a problem that:

  1. it contains a choice of kkk different actions in each step
  2. the reward of different actions have stationary probability distributions
  3. the main purpose of the actions is to maximize the cumulation of the reward of each step
  4. the whole experiment should be repeated at certain times like 1000.

These 4 elements constitute several tasks in the real world. And these 4 element is also meaningful for reinforcement learning research. Then some concepts of reinforcement learning are given from these 4 elements:

  1. ‘value of action’: it is the expected/mean reward of an action
  2. AtA_tAt​: action selected by the agent at time ttt
  3. RtR_tRt​: reward at time ttt corresponding to the action AtA_tAt​
  4. q⋆(a)q_{\star}(a)q⋆​(a): value of an arbitrary action aaa , and

    q⋆(a)≐E[Rt∣At=a]q_{\star}(a)\doteq \mathbb{E}[R_t|A_t=a] q⋆​(a)≐E[Rt​∣At​=a]

  5. Q(a)Q(a)Q(a): estimation of value of action aaa

However, for the most time the value of an action q⋆(a)q_{\star}(a)q⋆​(a) is unknown. And once it had been calculated, what we need to do is just to repeat the best action which had the greatest value. But almost all the time what we knew is just its estimation Q(a)Q(a)Q(a). There is also the biggest value among Q(a)Q(a)Q(a), but for it is just an expectation, the biggest value might be not q⋆q_{\star}q⋆​. So what we need to concern is how to find the greatest value. An intuitive idea is that for each step selecting the action who has the greatest value currently(this is called ‘exploitation’) or searching the action space to update Q(a)Q(a)Q(a) for all aaa (this is called ‘exploration’) are employed randomly. For a short time interval, exploitation can maximize the reward and this is also called greedy action, but for a long time, exploration can produce a maximum reward. Because the agent must select the exploitation or the exploration, so they are conflicting. How to balance them is a great issue in reinforcement learning.

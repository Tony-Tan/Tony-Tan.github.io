(draft)Finite Markov Decision

Posted on 2020-04-14 Edited on 2020-11-08 In category Disqus:

Keywords:

In the previous posts, we talked about the k-armed bandit task in stationary or non-stationary situation. And we also had introduced associative search and non-associative problems. The k-armed bandits are non-associative task but the full reinforcement learning is associative.[1]

Now, we take a step towards the full reinforcement learning task, we introduce finite Markov Decision Processes, finite MDP for short, which contains both evaluating feedback and an associative aspect that the agent choose action in different situations. In the bandit problem, action influence nothing but the immediate reward. However, in the finite MDP that we will investage, the action influence:

  1. immediate reward
  2. sequencial situation
  3. state
  4. future reward

In the bandit problems, which is nonassociative, the agents or the dicision makers only concern which action should be taken currently to maximize the total rewards. But to an associative task, not only immediate rewards but also delay rewards should be concerned. And what the agent and dicision maker should do in an associative task is to balance the delay rewards and the immediate rewards. In the post ‘Action-value methods’, we introduced the value function and how to estimate the value function, based on which we can select the action giving a higher total reward in a long run. And the value function is only decided by action:

q∗(a)q_{*}(a) q∗​(a)

However, in finite MDP, the value function is of both action and the state that the agent or decision maker faced. Then it converts to:

q∗(s,a)q_{*}(s,a) q∗​(s,a)

and for the state, now, is a relatively important element in the finite MDP, the associative task, we can study the relation between the states and the value given a determined policy basing on which the action would be selected. The value function of state is:

v∗(s)v_{*}(s) v∗​(s)

and it is essential to assign credit to individual action selection for long-term consequence, because the state transmission is decided by actions, and if we have known which state is better in the next step then we could choose an action to reach that state.

The finite MDP is not a full reinforcement learning, but is a good idealized mathematical model for full reinforcement learning. And some key elements of this kind of mathematical model are:

  1. returns
  2. value function
  3. Bellman equations

All we have talk is far from the real intelligence, for our basic tool is mathematics, and there is tension between breadth of applicability and mathematical tractability.

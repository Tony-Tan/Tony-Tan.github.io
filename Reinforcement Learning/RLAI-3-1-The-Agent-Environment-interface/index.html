(draft)The Agent-Environment interface

Posted on 2020-04-14 Edited on 2020-11-08 In category Disqus:

Keywords:

Now, we come into a new field called finite MDP(introduced in the post ‘Finite Markov Decision’). Before we investigate the detials of the finite MDP algorithms, we should make some concepts clear first. In this post we talked about the Agent-Environment interface.

Agent and Environment[^1]

Agent is the learner or decision maker who decides which action to take. And other things in the task is called environment with which the agent interact. In control theory, the agent, environment, and action are called controller, controlled system and control signal respectively. There are many ways through which the agent could interact with environment. In the k-armed bandits problem, the agent is the gambler, and the environment is all the other things, like the machine, the people next to him, the casino, and so on. However, in the k-armed bandits, we sacrifices almost all detials of the environment to simplify the task, and to simplify the task one step closer, we assume the distribution of each action is fixed – stationary condition. The action of the agent, the gambler, is the selection of arm at current time step. And the environment produces reward and all other things go back to the initial state.

But in full reinforcement learning, after each action or even just with the time fleeting, the situation or the state of environment is changing. Finite MDP is a little more complex than the k-armed bandit problem. Its state is changed after a certain action:

Beside the state is changed by each action in each step, other elements of finte MDP is almost the same as the k-armed bandits. And the process of finite MDP starts from initial state: $S_0$ and then an action should be selected, and the reward and next state $S_1$ come to the agent. So the sequence is:

$$
A_0\to A_0\to R_1\to S_1 \to A_1\to R_2\to S_2 \to A_2\to \cdots\tag{1}
$$

For the finite MDP, the number of steps is finite. And, of course, there is infinite MDP as well. But we not study it right now.

Mathematics of Finite MDP

The central concept of MDP and finit MDP is that at step $t$ the reward $R_t$ and state $S_t$ are considered as random variables who have well defined discrete probability distributions dependent only on the preceding state $S_{t-1}$ and action $A_{t-1}$ (for all steps that $t\geq 1$). And so we can define the probability distribution of current state and reward:
$$
p(s’,r|s,a)\doteq \mathrm{Pr}{S_t=s’,R_t=r|S_{t-1}=s,A_{t-1}=a}\tag{2}
$$

where $s’,s \in \mathcal{S}$, $r\in \mathcal{R}$ and $a\in \mathcal{A}(s)$. And $p$ is called dynamics of the MDP. It’s also a conditional probability: $\mathcal{S}\times\mathcal{R}\times\mathcal{S}\times\mathcal{A}\to[0,1]$

And $p$ described above completely characterizes the environment’s dynamics, for:

  1. probabiltiy of $S_t$ and $R_t$ depends only on $S_{t-1}$ and $A_{t-1}$
  2. states and actions from early steps, like step $t-2$ or earlier(where $t>2$), do not influnce current state and reward derictly.
  3. the decision process is not useful here, either. All things are rely on the state

If a state include information about all aspects of past agent-environment interaction that make a difference for the future. The state is said to have Markov Property.

From equation (2), many other mathematical properties could be deduced:

State-transition Probability

This probability is about the current state given preceding state and action:
$$
\begin{aligned}
p(s’|s,a)\doteq& \mathrm{Pr}{S_t=s’|S_{t-1}=s,A_{t-1}=a}\
=&\sum_{r\in\mathcal{R}}p(s’,r|s,a)
\end{aligned}
\tag{3}
$$

This distribution function $p$: $\mathcal{S}\times\mathcal{S}\times\mathcal{A}\to[0,1]$

Expected Reward for State-action Pairs

We can also derive the expected reward from equation(2):

$$
\begin{aligned}
r(s,a)&\doteq \mathbb{E}[R_t|S_{t-1}=s,A_{t-1}=a]\
&=\sum_{r\in\mathcal{R}}r\sum_{s’\in \mathcal{S}}p(s’,r|s,a)
\end{aligned}\tag{4}
$$

then we get a new function $r:\mathcal{S}\times\mathcal{A}\to \mathbb{R}$

Expected Rewards for state-action-next-state Triples

Our goal is to maximize the sum of reward of the whole process. Then we concern more about the rewards or expected rewards:

$$
\begin{aligned}
r(s,a,s’)&\doteq \mathbb{E}[R_t|S_{t-1}=s,A_{t-1}=a,S_t=s’]\
&= \sum_{r\in\mathcal{R}}r\frac{p(s’,r|s,a)}{p(s’|s,a)}
\end{aligned}\tag{5}
$$
then we get a new function $r:\mathcal{S}\times\mathcal{A}\times\mathcal{S}\to \mathbb{R}$

More Detials of Agent and Environment

Steps, Action and State

Some other properties of concepts are also needed to declare:

  1. the steps in the process is not necessary to refer to as fixed intervals of real time. It can be arbitrary successive stage of decission makeing and acting
  2. Action can be low-level, like the movement of robot’s arm, and high-level, like the decission of movement from a chess player. And it can be mental action, which can not be observed.
  3. State can be low-level, like the sensation, and high-level, like symbolic descriptions of object as well. And it also could be based on memory of past sensations or entirely mental or subjective.

Boundary between Agent and Environment

The boundary between agent and enviroment is not the same as the physical boundary of a robot or animal’s body, that is we consider musle, skeleton, sensory organs is part of environment. The environment cannot be changed by the agent arbitrarily. And agent knows something about the environment. The boundary between agent and environment represents the limit of the agent’s absolute control but it does not represents the knowledge of agent, such as the agent may know how the Rubik’s cube works but could not solve it.

Conclusion

MDP framwork is the abstraction of goal-directed learning from interaction problems of goal-directed behavior and it can be reduced to 3 signals: action, state and reward.
